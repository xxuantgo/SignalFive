  - M系列（2个，交易行为）：`M01` 日内-隔夜博弈差，`M03` 买卖压失衡比。
  - N系列（1个）：`N01` 纯特质动量（剔除当日截面均值）。

  - `F03` 期限利差因子（中长端-中短端，zscore）
  - `F04` 通胀剪刀差因子（生产端-消费端，zscore）
  - `F05` 大小盘估值比（当前在仓位层权重设为0）
  - `F06` 中美实际利差因子（zscore差）
- 预处理流程（截面因子）：先做截面去极值（winsorize），再做截面 rank 标准化（默认；薄截面下比原值和纯zscore更稳健）。

### 4.3 单因子测试

- 未来收益定义：`ret_{t,t+5}`（与周频持有期匹配）。
- 每个交易日做截面 Spearman 相关（Rank IC）。
- 统计输出：`mean_ic`, `std_ic`, `icir`, `p_value`, `win_rate`。

公式：

`IC_t = Spearman(rank(f_t), rank(r_{t,t+5}))`

`ICIR = mean(IC_t) / std(IC_t)`

### 4.4 防前视偏差（核心）

项目中有四层硬控制，其中“因子筛选不看未来”是最关键的一层：

1. 回测前样本筛选有效因子：以 `BACKTEST_START=2021-01-04` 为边界，筛选截止日设为 `2021-01-03`；实际使用的是 2019-11 至 2020-12（含边界前交易日）的因子序列与未来5日收益来计算 IC/ICIR，再筛选有效因子，不使用回测期（2021+）信息。
2. `IC[t]` 本身依赖 `t->t+5` 收益，因此在用于筛选/定向/合成前统一 `shift(5)`，把“可观测时点”对齐到真实可用信息时刻。
3. 调仓日使用“前一交易日”信号，优化窗口收益也严格截止到 signal date，避免读到调仓日收盘后的数据。
4. 严格 OOS 流程中，每个 outer 段都只用该段开始前历史做调参（`tune_end = outer_start-1交易日`），参数不跨段泄露未来信息。

补充口径说明：提交版 `run_main.py` 为了复现稳定，最终使用 `base.py` 内置固定有效因子清单；该清单来源于上述“回测前样本筛选 + 去重”流程，而不是在回测区间内动态重筛。

### 4.5 因子合成（icir_robust）

在滚动 ICIR 动态权重上叠加三层鲁棒化：

1. 回测前质量分（|ICIR| + 显著性）
2. 相关性惩罚（降低高相关冗余）
3. 时间平滑（EWMA 抑制权重抖动）

合成后再次做截面 rank，得到 `composite_score ∈ [0,1]`。

### 4.6 组合优化（hybrid_cvar_rp + strict OOS Bayes）

#### 4.6.1 组合优化常见方法（文献与实务）

在资产配置问题中，常见方法包括：
- 均值-方差（Markowitz）：直接优化期望收益与方差，解释性强，但对收益预测误差和厚尾分布敏感。
- 最小方差（Min-Variance）：不显式预测收益，只压低组合波动，稳健性较高但可能忽视尾部风险。
- 风险平价（Risk Parity）：让各资产风险贡献趋于均衡，通常比均值-方差更稳，但在极端行情下未必能控制尾部损失。
- CVaR/ES 优化：直接最小化尾部损失（条件在险价值），适合非正态、厚尾收益分布。
- 鲁棒优化/分布鲁棒优化：考虑参数不确定性，代价是实现复杂度和计算量上升。

#### 4.6.2 本题特征与方法选择逻辑

本题有四个关键特征：
- 资产数量小（28只ETF），但风格跨度大，协方差结构会随阶段变化。
- 调仓频率周度，样本窗口有限，纯收益预测容易不稳定。
- 赛题强调风险调整收益（Sharpe）与回撤控制，尾部风险不能忽略。
- 存在交易成本与持仓约束（`w_i<=35%`、`N>=3`、非负权重）。

因此本项目采用“二阶段”思路：
- 第一步先由因子做 TopN 资产筛选（把“选什么资产”与“怎么配权重”分开）。
- 第二步在候选资产内做风险优化，不直接押注收益均值，核心用 `hybrid_cvar_rp`：
  - `CVaR` 负责尾部风险控制；
  - `RP` 负责分散化和稳健锚定；
  - 两者凸组合在“稳健 + 收益效率”之间折中, 对 ETF 这种相关性强、尾部共振明显的资产池，混合方法经常更实用

#### 4.6.3 数学形式与约束

在每个调仓日 `t`，对 TopN 资产求解权重 `w_t`，满足：
- `w_i >= 0`
- `sum_i w_i = 1`
- `w_i <= 0.35`
- `|{i: w_i>0}| >= 3`

核心组合为：

`w_t = (1-β) * w_t^CVaR + β * w_t^RP,  β∈[0,1]`

其中：
- `w_t^CVaR` 由历史收益窗口估计，支持三种方法：
  - `empirical`：Rockafellar-Uryasev 线性规划（历史场景CVaR）；
  - `parametric`：高斯参数化 CVaR；
  - `cornish_fisher`：偏度/峰度修正的参数化 CVaR。
- `w_t^RP` 通过最小化风险贡献离散度得到（SLSQP）。
- 两者混合后再做 capped-simplex 投影与约束后处理，确保可行性。

换手成本通过惩罚项进入 CVaR 子问题：

`penalty_turnover = λ * ||w_t - w_{t-1}||_1`

这样可抑制无效交易、提高实盘可执行性。 -- prev_weights + turnover_lambda 的意义是“让策略像真实交易一样”，不至于因为信号/统计噪声每天抖动导致频繁换仓。

#### 4.6.4 工程实现（与代码逐项对应）

实现路径：
- 候选资产选择：`backtest/engine.py::build_optimized_schedule`
  - 调仓日用前一交易日信号；
  - 收益窗口严格截止到 signal date；
  - 每次只在 TopN 上优化。
- 权重求解：`portfolio/optimizer.py`
  - `cvar_weights`（三种CVaR口径）
  - `risk_parity_weights`
  - `hybrid_cvar_rp_weights`
  - `_post_process_weights` + `_project_to_capped_simplex` 保障约束可行。

这保证了“同一选股信号下，权重优化口径一致且可复现”。

#### 4.6.5 Optuna 调参（最新做法：`run_strict_oos_stitch.py`）

本项目最新方案不是一次性全样本调参，而是严格时间推进的“外层OOS + 内层Bayes”：

1. 外层（outer fold）按时间分段，作为真正样本外测试段。
2. 每个 outer 段开始前，用 `tune_end` 之前历史构建内层 fold 做参数搜索。
3. 搜索空间（典型）：
   - `top_n`
   - `cvar_alpha`
   - `cov_window`
   - `turnover_lambda`
   - `hybrid_beta`
   - `cvar_method`
   - 代码默认范围（`run_strict_oos_stitch.py`）：`top_n∈[3,9]`，`alpha∈[0.92,0.96]`，`cov_window∈[60,220]`，`lambda∈[1e-3,3e-2]`（log采样），`beta∈[0.10,0.50]`。
   - 本次严格OOS运行（`run_strict.bat`）采用了更窄、更稳健区间：`top_n∈[3,7]`，`alpha∈[0.93,0.95]`，`cov_window∈[120,180]`，`lambda∈[3e-3,1.5e-2]`，`beta∈[0.05,0.25]`。
4. 目标函数（每个 outer 独立）：

`objective = mean_sharpe - a*std_sharpe - b*max(0, sharpe_floor-min_sharpe) - c*avg_turnover - insufficient_fold_penalty`

其中 `a,b,c` 分别对应 `obj-std-penalty / obj-worst-penalty / obj-turnover-penalty`。
另外，若内层有效折数不足，会额外扣罚（代码中按缺失折数线性惩罚），并将“无有效折”的候选直接赋极差目标值（`-1e6`）淘汰。

5. 首段稳态机制：`no_bayes_first_n_outer`
   - 前 N 个 outer 段可直接使用锚定参数（来自 `base.py`）；
   - 解决早期样本偏短导致的搜索不稳定问题。
6. 内层评估采用“训练段预热 + 测试段打分”：
   - 调仓路径从 inner fold 的 `train_start` 开始滚动；
   - 但 Sharpe 只在该 fold 的 `test_start~test_end` 统计；
   - 这样既保留真实持仓路径依赖，又不把训练段绩效混进评分。
7. 每段选出的最优参数只用于该段 OOS；最终再做净值首尾拼接，得到全区间严格 OOS 结果。

#### 4.6.6 为什么这种实现更可信

与“全样本一次调参”相比，当前 strict-oos Bayes 实现有三点优势：
- 时间因果严格：每段参数都只看该段开始前数据。
- 稳健目标优先：惩罚跨折波动、最差折与换手，不追求单次最优。
- 可审计：每个 outer 的内层折、trials、入选参数、分段绩效均落盘，可复核完整决策链。

## 7. 70% 逻辑展示的核心亮点

1. 工程闭环完整：单脚本可复现（`run_main.py`），严格样本外另有独立流程（`run_strict_oos_stitch.py`）。

## 8. 加分项建议（新优化思路，科学且可落地）

当前已实现的稳健目标是：

`mean(OOS Sharpe) - λ1*std(OOS Sharpe) - λ2*worst_fold_penalty - λ3*turnover - λ4*active_share_to_RP`

可进一步升级为“状态条件风险预算”框架（可作为加分项提案）：

1. 用 Regime 压力分数动态调 `CVaR alpha`（高压力 -> 更保守尾部约束）。
2. 用 Regime 压力分数动态调 `beta`（高压力 -> 提高 RP 权重，降低 CVaR 激进暴露）。
3. 在目标函数中新增“跨状态稳定性惩罚”：
   - 惩罚牛市/震荡/风险期之间 Sharpe 方差过大。

这相当于从“单一分布最优”升级到“跨市场状态稳健最优”，符合加分项的创新方向。
